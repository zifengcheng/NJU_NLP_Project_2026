# 从大语言模型中抽取长文本表示

\---

## 一、背景要求

当前，大语言模型在零样本自然语言处理领域展现了强大的语义理解能力，推动了文本表示技术的不断革新。从大模型中抽取长文本表示是一个具有潜力的方向，因为它**既不需要额外数据也不需要额外训练**。目前主要存在两种方法，第一种是利用提示工程引导大模型将长文本表示压缩到一个词元（token）。另外一种方法是对所有词元进行平均池化操作以聚合出长文本表示。本次大作业希望对比这两种方法，并对第二种方法进行进一步探索。

\---

## 二、基于提示工程的抽取方法介绍

基于提示工程的抽取方法主要依赖于设计精妙的提示模板，以引导大语言模型在解码过程中把文本表示压缩到最后一个词元。最经典的是PromptEOL（Prompt-based method with Explicit One word Limitation）为代表的语义压缩提示，该提示为PromptEOL：This sentence : ”\[text] ” means in one word:“。

## 三、任务描述

本次实验采用 **QMSum、2WikiMultihop、ArguAna**，作业任务分为三个任务。**除了特别说明外，所有实验均需报告三个数据集的结果**。模型为Mistral-Instruct-7B-0.3，如果跑不起来可以用Qwen2-1.5B-Instruct，不过会被扣分。具体实验内容如下：

### 3.1 简单任务（20分 = 结果准确性15分 + 报告质量5分）

* **任务内容**：
* **实验评估**：使用PromptEOL和mean-pooling抽取长文本表示进行评估。
* **探索大模型不同层的句子嵌入质量**：探索两者使用不同抽取层时对表示的影响。
* **提示模板参考**，其中text是句子的占位符：

  * PromptEOL：This sentence : ”\[text] ” means in one word:“



### 3.2 分析任务 （20分 = 准确性15分 + 报告质量5分）

请完成以下所有分析实验：

1. 对比PromptEOL和mean-pooling的结果，并分析两者中性能更好的方法为何更有效。
2. 了解大语言模型的RoPE位置编码，探索RoPE中频率对结果的影响，并分析为什么。

### 3.3 进阶任务 （60分 = 创新性30分 + 实验结果20分 + 报告质量10分）

请考虑当前研究的不足，创新性地提升性能。考虑至少三个研究点来综合提升性能，最后报告自己的创新点及对应的实验表现。三个研究点都要可以对性能产生提升，且至少其中两个研究点是可以结合在一起的。

我们提供了以下研究点：

1. 由于长文本中词元较多会影响语义传递，怎么放大文本中的关键词元对整体效果的影响。
2. 怎么选择长文本中的关键词元进行聚合以提升效果。
3. 语义压缩角度。尝试对长文本进行语义压缩后，再进行编码。
4. 是否可以通过分而治之的思想对长文本切分后再聚合。
5. LLM中是否可以存在一个子结构/子神经元 适配长文本表示。该研究点应该有清晰的动机，比如为何是这些子结构和神经元。

注：
1. 使用词性标注工具等，让the、a、an这类token不参与mean pooling的计算不得分。
2. 利用3.1的layer提升性能不算研究点。利用3.2的RoPE来提升性能一般也不算，除非有针对性的深入的研究。
