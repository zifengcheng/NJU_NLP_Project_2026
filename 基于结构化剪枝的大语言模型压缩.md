# 基于结构化剪枝的大语言模型压缩

## 一、背景介绍

当前，大语言模型在多个复杂自然语言处理任务中展现出强大能力，但其参数规模大、推理成本高，给实际部署带来了显存占用、计算开销和响应延迟等问题。因此，如何在尽量保持模型性能的同时压缩模型规模，是大语言模型应用中的重要问题。

结构化剪枝是一种有效的模型压缩方法，其核心思想是删除模型中冗余的完整结构单元。目前，常见的结构化剪枝方法主要包括两类：第一类是对多头注意力模块进行剪枝，移除贡献较小的 attention heads；第二类是对前馈神经网络进行剪枝，裁剪 FFN 中间神经元或隐藏维度。

本次大作业的目标是实现并对比上述两类结构化剪枝方法，分析不同剪枝策略对模型性能、参数量和推理效率的影响，并在此基础上对剪枝技术进行进一步探索。

## 二、任务描述

推荐模型包括 meta-llama/Llama-3.2-3B-Instruct、Qwen/Qwen2.5-3B-Instruct、zai-org/chatglm-6b。
其他开源大语言模型也可以使用，但需在报告中说明模型信息及选择原因。

推荐数据集包括 MMLU、HellaSwag、ARC-Easy、ARC-Challenge 和 OpenBookQA，可根据计算资源选择其中一个或多个数据集完成实验。

### 3.1 简单任务（20分 = 结果准确性15分 + 报告质量5分）

- **任务内容：** 实现基础的结构化剪枝方法，并完成剪枝前后的性能对比。
- **实验评估：** 至少实现以下两种结构化剪枝方法：
    - **Attention Head Pruning：** 对多头注意力模块进行剪枝，移除部分贡献较小的 attention heads。
    - **FFN Neuron Pruning：** 对前馈神经网络进行剪枝，裁剪 FFN 中间神经元或隐藏维度。
- **剪枝比例设置：** 至少设置三个剪枝比例，例如 10%、30%、50%，并报告不同剪枝比例下的实验结果。
- **评价指标：** 需报告剪枝前后模型的任务性能、参数量、模型大小和推理效率。
- **基础分析：** 比较 attention head pruning 和 FFN pruning 在性能下降、参数减少和推理加速方面的差异。

### 3.2 分析任务（20分 = 准确性15分 + 报告质量5分）

请至少完成两个以下的分析实验：

- **对比两类结构化剪枝方法的效果：** 对比 Attention Head Pruning 和 FFN Neuron Pruning 的实验结果，并分析哪一种方法在相同剪枝比例下对模型性能影响更小，哪一种方法对参数量或推理效率改善更明显。
- **分析不同层的剪枝敏感性：** 分析 Transformer 不同层对剪枝的敏感程度。例如，可以分别对浅层、中间层和深层进行剪枝，观察不同层被裁剪后模型性能的变化，并分析哪些层更适合剪枝。
- **分析剪枝后恢复训练的作用：** 比较“直接剪枝”和“剪枝后继续微调”两种设置下的实验结果，分析恢复训练是否能够缓解剪枝带来的性能下降。
- **分析参数减少与实际加速之间的关系：** 讨论剪枝后模型参数量减少是否一定会带来推理速度提升，并结合实验结果分析可能原因。

### 3.3 进阶任务（60分 = 创新性30分 + 实验结果20分 + 报告质量10分）

请考虑当前结构化剪枝方法的不足，创新性地提升模型压缩效果。
请至少设计**三个研究点**来综合提升剪枝后的模型性能或效率，并报告每个研究点的动机、方法设计和实验表现。
三个研究点都应具有明确实验结果支撑，且至少其中两个研究点可以结合在一起，形成一个完整的改进方案。

我们提供以下可选研究点，其他研究点也可自行选择：
- **任务感知的重要性评分方法：** 基础剪枝方法通常只根据权重范数或随机策略选择被剪枝结构。可以考虑结合任务数据，设计更合理的重要性评分方法。例如，综合权重范数、激活值、梯度信息或损失变化来判断 attention head 或 FFN neuron 的重要性。
- **层感知的自适应剪枝率分配：** 不同 Transformer 层对模型性能的贡献可能不同。可以探索不同层使用不同剪枝率，而不是所有层采用统一剪枝比例。例如，对剪枝敏感层保留更多结构，对冗余层进行更大幅度裁剪。
- **Attention Head 与 FFN 的联合剪枝策略：** 单独剪枝 attention head 或 FFN neuron 可能无法充分压缩模型。可以设计联合剪枝方法，在给定总剪枝率、参数量或推理延迟约束下，同时选择需要裁剪的 attention heads 和 FFN neurons。
- **渐进式剪枝策略：** 一次性剪枝可能导致模型性能明显下降。可以尝试多阶段渐进式剪枝，例如每次剪枝一小部分结构，并在每个阶段后进行短暂恢复训练，分析其相较于一次性剪枝的优势。
- **剪枝后恢复训练或参数高效恢复方法：** 可以探索剪枝后如何更有效地恢复模型性能。例如，比较全量微调、LoRA 微调、只微调分类头、只微调 LayerNorm 等不同恢复策略的效果。
- **面向鲁棒性的剪枝分析：** 除了关注标准测试集上的性能，还可以构造扰动输入，例如随机删除词、同义词替换、拼写扰动等，分析剪枝后模型鲁棒性是否下降，并尝试设计更稳健的剪枝策略。
- **面向效率约束的剪枝方案：** 不只以剪枝比例为目标，而是以实际推理延迟、显存占用或吞吐量为约束，设计更符合部署需求的剪枝方案。例如，在性能下降不超过 2% 的前提下最大化推理速度提升。
